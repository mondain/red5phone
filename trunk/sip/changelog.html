Red5Phone Changelog

0.0.04 - 11th July 2009

Dele Olajide contributions
  • Now compatible with Red5 0.8.0.
  • Incoming calls still not working properly. Sometimes mjsip gets stuck on "Trying". Any help appreciated.
Lior Herman contributions
  • Outbound proxy support added.

0.0.03 - 25th March 2009

Dele Olajide contributions
  • Fixed DTMF. Tested with UK PSTN and works ok.
  • Added Nellymoser encoder from Joseph Artsimovich (jart@users.sourceforge.net)
Mauro Brasil, Rafael and UOL (www.uol.co.br) contributions
  • Cleaned up the code
  • Implemented multi-codec support for Red5phone (PCMU, PCMA, iLBC and G.729). G.729 requires payment of royalities and license rights if used.
  • Adjustments to support different packetization were made as well.
Prabhu Tamilarasan contributions
  • Normalize function to improve volume
Lior Herman contributions
  • Red5phone show busy or rejected message for busy outgoing calls.
  • Openlaszlo and Javascript API made compatible with latest chages to Red5phone
  • Add Dial external command to Flex to dial from a link.
  • Used UID (unique identifier) in Flex instead of the username value for binding the sipprovider object in Mjsip. Now you can use same sip account to register from multiple remote locations.
  • Changed registration of red5phone using phone@sip_provider.getViaAddress() instead of phone@realm
  • Mjsip now uses Outboundproxy = Proxy like that all sip headers using realm, but the message is send to the proxy ip.
  • Process of OPTIONS messages in Mjsip some sbc’s using to check call keep alive.
  • Fix authentication header for REG and INVITE to use cnounce value when proxy send qop= auth in 401 or 407 message.
  • Fix red5phone application close for complete unregister before sipprovider.halt is execute.
  • Fix the microphone.init in Flex to not be bind to the REGISTER SUCCESS message as it can be many SUCCESS messages coming during a call and mic need to be init only once.
  • Fix Cancel message for Invite (call) that have not been answered yet. When you want hangup a call before the remote user answered Mjsip send wrong Cseq in Cancel message.
  • Fixed NullException error when application is closed in provider.halt() its try to close a null tcp_socket and get Null exception back. Fix is in org.zoolu.net.TcpServer.java in the end of the file.
Janny_buh contribution:
  • Patch all sip Register message using same Call-ID header and increment Cseq as recommended by RFC3261.

0.0.02 - 4th January 2009

  • Replaced aso2ulaw with java implementation of asao codec decoder.
  • Fixed delay in red5phone voice.
  • Red5phone username is no more used for SIP contact details. Phone number is now used instead.
  • Fixed fault with Red5Phone not responding to 407 requests
  • DTMF not sent when there is silence. Many thanks to Pierrick Grasland for finding and fixing the fault.
  • Asterisk Voice Mail for Red5phone. More Asterisk features to come.
  • Autologin for Red5phone. See red5phone.html
  • Fixed incoming call failure with Red5phone.
  • New user Interface for Red5phone. Many thanks to community member Naoki for doing this.
  • Javascript and Openlaszlo libraries for Red5phone.
  • New "login" and "register" methods in Red5phone to allow login without SIP registration.
  • Added DTMF to SparkWeb SIP Phone.
  • Added Sound effects to DTMF.
  • Fixed fault with sip user set to "user" instead of supplied username. It now enables users registered with a proxy to make calls without requiring proxy authentication.
  • Added important note to readme for Linux users of Red5Phone. Many thanks to Chris to testing with Linux.
  • Improved voice quality by removing ADPCM compression. It can be changed from admin console.
  • Implemented DTMF - RTC2833. New remote method "dtmf".

0.1.01 - 1st May 2008

  • Initial Release of Red5Phone
